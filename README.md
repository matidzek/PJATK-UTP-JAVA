## Assignments

### UTP2 – Lambdas & Functional Patterns

| Task | Classes | Description |
|------|---------|-------------|
| zad1 | `Maybe`, `Main` | Null-safe container `Maybe<T>` with `of`, `ifPresent`, `map`, `get`, `isPresent`, `orElse`, `filter` |
| zad2 | `InputConverter`, `Main` | Generic converter with a single `convertBy(functions...)` method for composing arbitrary chains of `Function`s |

### UTP3 – Collections & Maps

| Task | Classes | Description |
|------|---------|-------------|
| zad1 | `XList`, `Main` | Extended `ArrayList` with `union`, `diff`, `unique`, `combine`, `collect`, `join`, `forEachWithIndex`, factory methods `charsOf` / `tokensOf` |
| zad2 | `Purchase`, `CustomersPurchaseSortFind`, `Main` | Reads `~/customers.txt`; sorts purchases by surname and by cost; filters by customer ID |
| zad3 | *(ProgLang)* | Reads `~/Programmers.tsv`; builds language→programmers and programmer→languages maps with sorting and filtering via generic `sorted()` / `filtered()` helpers |

### UTP4 – Streams & File I/O

| Task | Description |
|------|-------------|
| Anagrams | Finds all anagram groups from `unixdict.txt`; prints the group(s) with the most anagrams; **requires Java Streams** |
| Finder | Reads `~/Test.java` and counts `if` statements and occurrences of `"wariant"` – ignoring comments and string literals |
| Futil | Recursively walks `~/UTP6dir/`, reads all `.txt` files (CP1250) and writes merged content to `UTP6res.txt` (UTF-8) using `FileVisitor` |

### UTP5 – Concurrency

| Task | Classes | Description |
|------|---------|-------------|
| zad1 | `StringTask`, `Main` | Long-running string concatenation task with states: `CREATED`, `RUNNING`, `ABORTED`, `READY`; supports `start()` and `abort()` |
| zad2 | `Main` | Swing GUI with `JList` showing running tasks as `Future`; supports querying state, cancelling, and viewing results |
| zad3 | `Towar`, `Producent`, `Konsument`, `Main` | Two threads cooperating – thread A reads goods from `../Towary.txt`, thread B sums weights; both print progress to console |

### UTP6 – Internationalisation (S_INTERDB)

Travel agency receives offer files from contractors in various locales (`pl_PL`, `en_GB`, …). Each line contains tab-separated: locale, country, departure/return date, destination type, price, currency.

| Class | Description |
|-------|-------------|
| `TravelData` | Parses offer files from `data/`; `getOffersDescriptionsList(locale, dateFormat)` returns offers translated and formatted for the target locale |
| `Database` | Stores all offers in a DB (`createDb()`), opens a `JTable` GUI with locale selector (`showGui()`) |
